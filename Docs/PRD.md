# FreeShot 产品需求文档（PRD）

---

## 1. 产品概述

| 项目 | 内容 |
|------|------|
| **产品名称** | FreeShot |
| **产品定位** | 免费 macOS 截图 / 录屏工具，面向内容创作者与演示场景 |
| **核心价值** | 免费、开源、本地优先，尽可能提供接近商业录屏工具的核心体验 |
| **目标平台** | macOS 13.0+ |
| **开源协议** | MIT |
| **目标用户** | 自媒体创作者、开发者演示、在线教育者、产品演示人员 |

### 1.1 产品目标

FreeShot 的目标不是做一个“功能很多但不稳定”的 demo，而是做一个**能在真实 macOS 使用场景里完成截图与录屏主流程**的工具。

现阶段优先级：

1. **先把截图/录屏主链路做稳**
2. **再补强增强能力与导出体验**
3. **最后再考虑云上传、分享等外围能力**

### 1.2 对标产品

- CleanShot X
- Shottr
- Kap
- Loom
- Screen Studio

### 1.3 差异化定位

FreeShot 的差异化方向：

- **免费 + 开源**
- **本地优先，不依赖 SaaS 才能使用核心能力**
- **优先满足中文用户与自媒体创作者的截图/录屏需求**
- **聚焦“截图 + 录屏 + 轻量后处理”的一体化体验**

## 4. 当前阶段核心问题（必须解决）

以下问题不是“优化项”，而是当前版本能否稳定使用的关键：

### 4.1 P0 问题

1. **录屏黑屏 / 输出异常**
   - 直接影响录屏主功能可用性
   - 必须优先修复 AssetWriter 与帧写入链路问题

2. **区域录屏坐标系统错误**
   - AppKit / CG 坐标系不一致会导致录制区域偏移
   - 多显示器与 Retina 下风险更高

3. **暂停逻辑不真实**
   - UI 显示暂停，但底层仍继续写帧会导致结果错误
   - 必须保证暂停行为与导出结果一致

4. **重复录制崩溃 / 状态未清理**
   - 一次录完再录崩溃会直接毁掉基本使用体验
   - 必须在会话结束后完整清理状态

5. **录屏文件生成稳定性**
   - 偶发写入失败、导出失败、时间轴异常都属于主链路故障

### 4.2 P1 问题

1. **多显示器表现不稳定**
2. **Retina 缩放与坐标映射不一致**
3. **权限引导不清晰，失败提示不够明确**
4. **连续录制时的稳定性不足**

## 5. 用户主流程

### 5.1 截图主流程

```text
状态栏图标 / 快捷键 → 选择截图模式（区域 / 窗口 / 全屏）→ 截图 → 预览 / 标注 / 保存 / 进入历史
```

### 5.2 录屏主流程

```text
状态栏图标 / 快捷键 → 选择录屏模式（区域 / 全屏）→ 可选倒计时 / 摄像头 / 键盘 / 鼠标 → 开始录制 → 暂停 / 继续 / 停止 → 输出 MP4 → 可选裁剪 / GIF 导出
```

### 5.3 当前体验要求

用户至少要能稳定完成：

1. 发起截图
2. 发起录屏
3. 停止录屏
4. 成功拿到可播放文件

如果这个主路径不稳，其他增强功能一律不能视作产品完成。

## 6. 技术方案

### 6.1 技术栈

| 层级 | 技术 |
|------|------|
| **UI** | SwiftUI + AppKit |
| **屏幕采集** | ScreenCaptureKit |
| **摄像头** | AVFoundation |
| **视频写入** | AVAssetWriter |
| **音频链路** | AVFoundation / AVAudioEngine |
| **OCR** | Vision / 本地识别链路 |
| **项目配置** | Xcode + XcodeGen |

### 6.2 技术重点

当前技术重点不是“继续堆功能”，而是：

- 统一 AppKit / CoreGraphics 坐标系统
- 稳定处理首帧时间戳与 AssetWriter session
- 规范录制状态机与清理逻辑
- 完善多显示器 / Retina 下的区域映射
- 建立可重复执行的真机测试清单

## 8. 验收标准

### 8.1 截图验收

- 区域 / 窗口 / 全屏截图均能稳定完成
- 截图结果区域正确，无明显偏移
- 预览、保存、历史可正常使用

### 8.2 录屏验收

- 区域 / 全屏录屏都能稳定输出可播放 MP4
- 不出现高频黑屏 / 空文件 / 无法停止问题
- 暂停后恢复的结果与预期一致
- 连续多次录制不崩溃
- 权限未开启时，用户能得到明确提示

### 8.3 多显示器 / Retina 验收

- 录制区域与用户选择区域一致
- 主副屏切换不导致坐标错乱
- Retina 缩放下不出现明显偏移

---

## 9. 当前不应过度承诺的内容

以下内容在完全验证前，不应在外部宣传中写成“稳定可用”：

- 滚动截图
- 多显示器完整支持
- 系统声音录制
- 云上传
- 长时间连续录制稳定性

这些能力可以写“已开发 / 开发中 / 待验证”，但不应假装已经完全成熟。

---

## 10. 开源与产品原则

FreeShot 当前的产品原则：

- **免费可用**：核心截图与录屏能力优先免费开放
- **开源透明**：问题、计划、进度尽量与代码状态一致
- **本地优先**：核心流程不强依赖云服务
- **先稳定后炫技**：优先把主链路做稳，再打磨外围能力

---

## 11. 结论

FreeShot 当前已经不是“从 0 到 1”的概念验证项目，而是一个**已经完成大部分功能开发、正在进入录屏稳定性攻坚阶段**的 macOS 工具项目。

当前最重要的产品目标不是继续加功能，而是：

> **把截图 / 录屏主链路打磨到可稳定交付。**
