# FreeShot 开发优化与验证计划

## 目标
先把核心录屏链路做成真正可用的 MVP，再逐步补齐截图、扩展功能和发布体验。

## Phase 1 - 核心录屏修复（已开始）
- 修复屏幕录制权限检查与引导
- 修复区域录制坐标映射
- 修复倒计时未生效问题
- 修复录制参数与 UI 选项脱节问题
- 为失败场景补充可见错误提示

## Phase 2 - 核心截图与功能绑定
- 修复区域截图和区域录屏复用同一选择器造成的行为串线
- 统一截图/录屏选区逻辑
- 校验截图预览、历史记录、保存链路

## Phase 3 - 稳定性验证
- 全屏录屏：开始 / 停止 / 保存 / 播放
- 区域录屏：选区 / 停止 / 保存 / 范围正确性
- 连续录制 2~3 次
- 有无画面 / 音频 / 空文件 / 黑屏
- 摄像头叠加 / 键盘显示 / 系统音频开关

## Phase 4 - 体验完善
- 权限文案更明确
- 异常提示从 console 提升到 UI
- README 中只保留真实已验证能力

## 当前已落地改动
- 录屏前先做屏幕录制权限与辅助功能权限检查
- 倒计时开始逻辑正式接入录屏流程
- UI 面板配置已绑定到 RecordingManager
- 录制区域按显示器坐标换算到 ScreenCaptureKit 的 sourceRect
- 区域截图与区域录屏已拆分各自回调，避免串线
- 录制失败会弹出警告框，而不是只在控制台 print

## 剩余必须在 macOS 真机完成的验证
- xcodebuild 编译
- 首次权限授权流程
- ScreenCaptureKit 真机录制
- 音视频文件完整性
- 多显示器与 Retina 行为
